# Концепция

Представляет из себя базовый класс для работы с контекстами. Контексты проходят через middlewareopen in new window, позволяя себя модифицировать.

У каждого контекста есть описание типов вызываемых событий

  • typeopen in new window — основной тип контекста (например message)
  • subTypesopen in new window — подтипы контекста (например [message_new, chat_kick_user])

Для удобного определения контекста, есть метод context.is()open in new window, позволяет искать сразу по типу и подтипу. Производит поиск пока не найдёт одно подходящее значение.

// Context { type: 'message', subTypes: [...] }
context.is(['message']); // => true

// Context { type: 'message', subTypes: ['chat_invite_user_by_link'] }
context.is(['chat_invite_user', 'chat_invite_user_by_link']); // => true

В каждом контексте, так же есть стандартное пространство для хранения простых состояний, в свойстве stateopen in new window

updates.on('message_new', async (context, next) => {
	context.state.user = await fetchUser(context.senderId);
	// При этом ничего вам не мешает расширить сам контекст
	// context.user = await fetchUser(context.senderId);

	return next();
});
